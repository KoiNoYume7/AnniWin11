# Architecture

Technical architecture overview for AnniWin11.

---

## Overview

AnniWin11 is a PowerShell 7+ automation suite for Windows 11 post-install setup. It follows a two-phase model per application: **install** (via winget/internet/local/terminal) then **config restore** (raw file copy from backup store).

## Entry Point

```
setup.bat
  -> UAC elevation check
  -> PowerShell 7 bootstrap (install if missing)
  -> Launch src/Main.ps1 via pwsh
```

## Directory Layout

```
AnniWin11/
  setup.bat              Entry point (batch, handles elevation)
  src/                   User-facing scripts (orchestration, features)
  lib/                   Reusable modules and helpers (project-agnostic where possible)
  config/                Configuration files (examples committed, generated files gitignored)
  docs/                  Documentation
  logs/                  Runtime log files (gitignored)
```

## Module Architecture

### lib/AnniLog.psm1

Reusable logging module. No AnniWin11-specific logic.

- **State**: module-scoped hashtable (`$script:AnniLogState`)
- **Levels**: ERROR (0), WARNING (1), INFO (2), SUCCESS (2), DEBUG (3)
- **Output**: coloured console via `Write-Host` + timestamped append to log file
- **Lifecycle**: `Initialize-AnniLog` -> `Write-AnniLog` (repeated) -> `Close-AnniLog`
- **Stopwatch**: optional, started on init, elapsed time logged on close

### lib/AnniLogo.psm1

Reusable ASCII art renderer. No AnniWin11-specific content.

- **Input**: array of strings (`$AsciiLines`), start/end colour hashtables
- **Rendering**: per-character ANSI 24-bit colour escape sequences
- **Gradient**: horizontal linear interpolation between two RGB colours

### lib/Config.ps1

Dot-sourced helper (not a module). Provides:

- **`Get-ProjectRoot`** -- resolves repo root from `$PSScriptRoot`
- **`Get-ConfigPath`** / **`Get-LogPath`** -- resolve paths relative to project root
- **`Read-JsoncFile`** -- strips JSONC comments (block + line + inline) then parses
- **`Read-JsonFile`** -- plain JSON reader
- **`Resolve-BackupPath`** -- expands path tokens (`appdata`, `localappdata`, `programdata`, `absolute`) to full filesystem paths

## Config File Convention

| File | Format | Committed | Purpose |
|------|--------|-----------|---------|
| `*_example.jsonc` | JSONC | Yes | Documented templates with comments |
| `*.json` | JSON | No (gitignored) | Runtime config, generated by `GenerateConfigs.ps1` |
| `backup_store.json` | JSON | No | Persists user-chosen backup drive path |
| `backup_manifest.json` | JSON | No (in backup store) | Timestamps and inventory of last backup |

## Script Responsibilities

| Script | Role |
|--------|------|
| `Main.ps1` | Interactive menu, orchestrates all other scripts |
| `InstallApps.ps1` | Reads `apps.json`, installs via winget/internet/local/terminal |
| `WinSettings.ps1` | Reads `settings.json`, applies registry-based Windows settings |
| `Pin-TaskbarApp.ps1` | Reads `taskbar_apps` from `settings.json`, generates LayoutModification.xml |
| `BackupConfigs.ps1` | Reads `app_configs.json`, copies app configs to backup store |
| `RestoreConfigs.ps1` | Reads `app_configs.json` + backup manifest, restores configs post-install |
| `DetectApps.ps1` | Compares `winget list` against `apps.json`, prompts user to add missing |
| `DriveSetup.ps1` | Interactive drive selector, writes `backup_store.json` |
| `GenerateConfigs.ps1` | Interactive config file generator from example templates |
| `BootstrapPwsh7.ps1` | Checks for / installs PowerShell 7, relaunches script under pwsh |

## Two-Phase App Lifecycle

```
Pre-reinstall:  BackupConfigs.ps1  -> snapshot config files to backup store
Post-reinstall: InstallApps.ps1    -> install software
                RestoreConfigs.ps1 -> copy config files back from backup store
```

## Security Considerations

- **DPAPI**: Browser saved passwords and session cookies are encrypted with machine-specific keys. They cannot be restored across Windows reinstalls. This is documented and warned at runtime.
- **UAC**: `setup.bat` requests elevation. Scripts that modify system-level settings (device name, dev mode) require admin rights.
- **No credential storage**: AnniWin11 never stores or transmits passwords, API keys, or session tokens.

## Path Token System

Used in `app_configs.json` to define where each app stores its config:

| Token | Resolves to |
|-------|-------------|
| `appdata` | `%APPDATA%` (typically `C:\Users\X\AppData\Roaming`) |
| `appdata_roaming` | Same as `appdata` |
| `localappdata` | `%LOCALAPPDATA%` (typically `C:\Users\X\AppData\Local`) |
| `programdata` | `%PROGRAMDATA%` (typically `C:\ProgramData`) |
| `absolute` | Literal path, used as-is |
